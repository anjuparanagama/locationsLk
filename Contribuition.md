# Contribuition

Thank you for contributing! This document explains how to contribute to this project.

## How to contribute

- Fork the repository.
- Create a feature branch: `git checkout -b feature/short-description`.
- Run linters and tests locally before committing.
- Open a pull request with a clear description and issue reference.

## Code style

- Follow existing TypeScript style and formatting.
- Use semantic commit messages (Conventional Commits recommended).

## Testing

- Run the test suite with your package manager (example):

```
pnpm test
```

## Reviewing

- Add a concise PR description, list the changes, and any manual testing steps.
- Keep PRs focused and small when possible.

## Contacts

- Open an issue if you're unsure where to start.
